# data-insight

产品数据分析：定义指标、设计埋点、解读数据。

## 输入

以下任意一种：
- 功能描述，需要定义衡量指标
- 已有数据，需要分析和解读
- 需要设计埋点方案

## 输出

结构化的数据方案或分析报告。

## 执行步骤

1. 确定分析目标：
   - 定义指标 → 梳理核心指标体系
   - 分析数据 → 解读趋势和异常
   - 设计埋点 → 生成埋点文档

2. **指标定义**模式：
   ```markdown
   ## [功能名称] 指标体系

   ### 核心指标（North Star）
   - [核心指标名]: [定义] | 目标值: [x]

   ### 过程指标
   | 指标 | 定义 | 计算方式 | 数据来源 | 查看频率 |
   |------|------|----------|----------|----------|
   | 转化率 | 完成操作的用户/访问用户 | count(完成)/count(访问) | 埋点 | 日 |

   ### 指标看板结构
   [建议看板布局：顶部核心指标 → 趋势图 → 明细表]
   ```

3. **埋点设计**模式：
   ```markdown
   ## 埋点方案

   ### 页面曝光
   | 事件名 | 触发时机 | 参数 | 说明 |
   |--------|---------|------|------|
   | page_view_home | 进入首页 | user_id, source | |

   ### 用户操作
   | 事件名 | 触发时机 | 参数 | 说明 |
   |--------|---------|------|------|
   | btn_click_submit | 点击提交按钮 | user_id, form_type | |

   ### 漏斗定义
   [步骤1: 访问页面] → [步骤2: 开始操作] → [步骤3: 完成提交]
   ```

4. **数据解读**模式：
   - 描述数据趋势
   - 指出异常点及可能原因
   - 给出行动建议

## 注意事项

- 指标不要贪多，核心指标 1-2 个，过程指标 5-8 个
- 每个指标必须有明确的计算方式，避免歧义
- 埋点命名统一用 snake_case，按模块前缀分组
- 区分虚荣指标和可行动指标，优先关注后者
