# prototype-spec

生成页面交互说明文档，描述界面布局、状态流转和交互逻辑。

## 输入

用户描述页面功能或交互需求。

## 输出

结构化的交互说明文档，包含页面结构、状态流转、异常处理。

## 执行步骤

1. 明确页面/流程范围：
   - 涉及哪些页面
   - 主要用户操作是什么
   - 有哪些状态切换
2. 对每个页面生成交互说明：

   ```markdown
   ## 页面：[页面名称]

   ### 页面结构
   ```
   ┌─────────────────────────┐
   │ Header: [导航栏说明]      │
   ├─────────────────────────┤
   │ [区域 1 说明]             │
   │ [区域 2 说明]             │
   │ [区域 3 说明]             │
   ├─────────────────────────┤
   │ Footer: [底部说明]        │
   └─────────────────────────┘
   ```

   ### 组件清单
   | 组件 | 类型 | 状态 | 交互行为 |
   |------|------|------|----------|
   | 搜索框 | Input | 默认/聚焦/加载中 | 输入后 300ms 防抖搜索 |
   | 列表 | List | 空/加载中/有数据/错误 | 下拉加载更多 |

   ### 状态流转
   [初始状态] → 用户操作 → [目标状态]

   ### 异常处理
   | 场景 | 处理方式 |
   |------|---------|
   | 网络错误 | 显示重试按钮 |
   | 数据为空 | 显示空状态引导 |
   | 权限不足 | 跳转登录页 |
   ```

3. 描述页面之间的跳转关系
4. 标注需要动效/过渡的地方
5. 展示给用户确认

## 注意事项

- 用 ASCII 示意图描述布局，比纯文字更直观
- 每个可交互组件都要说明所有状态（默认、悬停、点击、禁用、加载、错误）
- 异常状态和边界情况比正常流程更重要，一定不要遗漏
- 标注哪些行为需要二次确认（删除、支付等不可逆操作）
- 移动端和桌面端行为有差异时要分别说明
