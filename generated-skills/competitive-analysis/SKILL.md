# competitive-analysis

结构化的竞品分析，对比功能差异，提炼可借鉴的方案。

## 输入

用户提供竞品名称、要分析的功能点，或者竞品截图/链接的描述。

## 输出

结构化的竞品分析报告。

## 执行步骤

1. 确认分析范围：
   - 哪些竞品（2-5 个为宜）
   - 重点分析哪些维度（功能、体验、定价、技术方案）
2. 生成分析框架：

   ```markdown
   # [功能领域] 竞品分析报告

   ## 竞品概览
   | 维度 | 我们的产品 | 竞品A | 竞品B | 竞品C |
   |------|-----------|-------|-------|-------|
   | 核心定位 | | | | |
   | 目标用户 | | | | |
   | 上线时间 | | | | |

   ## 功能对比
   | 功能点 | 我们 | 竞品A | 竞品B | 竞品C | 备注 |
   |--------|------|-------|-------|-------|------|
   | 功能1 | ✅ | ✅ | ❌ | ⚠️ | |
   | 功能2 | ❌ | ✅ | ✅ | ✅ | 我们缺失 |

   ## 亮点分析
   ### 竞品A 的亮点
   - [具体亮点]: [为什么好，用户价值是什么]
   - 可借鉴程度：高/中/低
   - 实现成本评估：[S/M/L]

   ## 差距与机会
   ### 我们的优势
   - [优势1]
   ### 我们的差距
   - [差距1]: 建议 [行动方案]
   ### 差异化机会
   - [机会1]: [为什么这是机会]

   ## 行动建议
   | 优先级 | 行动项 | 参考竞品 | 预期效果 |
   |--------|--------|---------|----------|
   | P0 | | | |
   | P1 | | | |
   ```

3. 帮用户填充已知信息，标注需要进一步调研的部分
4. 给出优先级排序的行动建议

## 注意事项

- 分析要客观，不要为了突出自己而贬低竞品
- 功能有无只是表面，更重要的是分析背后的产品逻辑
- 竞品的做法不一定适合我们，要结合自身用户场景判断
- 标注信息来源和时效性，竞品随时在迭代
