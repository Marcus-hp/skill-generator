# 代码审查检查清单

## TypeScript 检查清单

### 安全性
- [ ] 没有硬编码的密钥、token、密码
- [ ] 用户输入有做验证和转义（XSS 防护）
- [ ] 没有使用 `dangerouslySetInnerHTML`（除非有 sanitize）
- [ ] API 请求有做认证和权限检查
- [ ] 没有暴露敏感信息到前端日志或错误信息

### 类型安全
- [ ] 没有滥用 `any` 类型（每个 any 都需要理由）
- [ ] 没有不安全的 `as` 类型断言（优先用类型守卫）
- [ ] 可能为 `null`/`undefined` 的值有做处理
- [ ] 泛型使用合理，没有过度复杂化
- [ ] 接口/类型定义导出给需要的消费方

### 性能
- [ ] 没有在循环里做 async/await（考虑 Promise.all）
- [ ] React 组件：大计算有用 useMemo，回调有用 useCallback
- [ ] React 组件：没有在 render 中创建新对象/数组作为 props
- [ ] 列表渲染有用合适的 key（不是 index）
- [ ] 没有不必要的重渲染（检查 useEffect 依赖）
- [ ] 大列表考虑虚拟滚动

### 错误处理
- [ ] async 操作有 try/catch 或 .catch()
- [ ] 错误有具体处理，不是空的 catch 块
- [ ] API 调用有处理超时和网络错误
- [ ] 用户看到的错误信息友好且有意义
- [ ] 关键操作有错误上报（Sentry 等）

### 代码质量
- [ ] 函数不超过 50 行（超过考虑拆分）
- [ ] 文件不超过 300 行（超过考虑拆模块）
- [ ] 变量命名有意义（不用 data, info, temp, result 等笼统名）
- [ ] 没有魔法数字（用常量替代）
- [ ] 没有重复代码（3 次以上出现就该抽象）
- [ ] 导入路径干净（没有 `../../../../../../`）

## Python 检查清单

### 安全性
- [ ] 没有硬编码的密钥、连接字符串
- [ ] SQL 查询用参数化而非字符串拼接
- [ ] 没有使用 `eval()` / `exec()`（除非绝对必要）
- [ ] 没有使用 `pickle` 反序列化不可信数据
- [ ] 文件操作有路径校验（防止路径穿越）
- [ ] 用户输入有做验证（Pydantic / marshmallow）

### 类型标注
- [ ] 函数参数和返回值有类型标注
- [ ] `Optional` 类型的参数默认值为 `None` 时有标注
- [ ] 复杂数据结构用 TypedDict 或 dataclass 定义
- [ ] 没有使用 `# type: ignore` 绕过检查（除非有注释说明原因）

### 性能
- [ ] 没有在循环中做数据库查询（N+1 问题）
- [ ] 大数据集用生成器而非一次性加载到内存
- [ ] 字符串拼接用 join 或 f-string，不用 + 号循环拼接
- [ ] 频繁查找用 set/dict 而非 list
- [ ] 数据库查询有合理的索引

### 错误处理
- [ ] 没有裸 `except:` 或 `except Exception:`（要捕获具体异常）
- [ ] 异常有记录日志（不是 print 或 pass）
- [ ] 资源获取用 `with` 语句（文件、连接、锁）
- [ ] 外部服务调用有超时设置和重试机制
- [ ] FastAPI/Django 有全局异常处理器

### 代码质量
- [ ] 函数职责单一（一个函数做一件事）
- [ ] 没有可变默认参数（`def f(lst=[])`）
- [ ] 类的 `__init__` 不做重逻辑（初始化归初始化）
- [ ] 导入按标准分组（stdlib → third-party → local）
- [ ] 没有 `import *`
- [ ] 测试覆盖了关键路径

## 通用检查清单

### 架构
- [ ] 变更符合现有架构模式（不要引入新范式除非有充分理由）
- [ ] 没有循环依赖
- [ ] 模块边界清晰（不跨层调用）

### 可维护性
- [ ] 新增代码有对应的测试
- [ ] 复杂逻辑有注释说明 why（不是 what）
- [ ] API 变更有更新文档
- [ ] 配置变更有更新 .env.example

### Git
- [ ] commit 粒度合理（一个 commit 一个目的）
- [ ] 没有提交 debug 代码（console.log / print / debugger）
- [ ] 没有提交被注释掉的代码块
- [ ] 没有提交生成文件（node_modules / __pycache__ / dist）
