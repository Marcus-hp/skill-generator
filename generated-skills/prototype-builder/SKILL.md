# prototype-builder

快速生成可交互的 HTML 原型页面或 UI 设计图。

## 输入

用户描述想要的页面或功能，可以是：
- "画一个登录页"
- "做一个后台管理的 dashboard"
- "电商首页原型"
- 或者提供一段 PRD / 交互说明

## 输出

可在浏览器直接打开的 HTML 原型文件，或 AI 生成的 UI 设计图片。

## 执行步骤

1. 确认原型需求：
   - 什么类型的页面（登录、列表、表单、Dashboard、详情页等）
   - 风格偏好（简约/后台/移动端/暗色主题）
   - 是否需要多页面跳转
   - 是否需要响应式（同时适配桌面和移动端）

2. 确认输出方式：
   ```
   你想要哪种形式的原型？

   1️⃣  HTML 可交互原型（可点击、可跳转、浏览器直接预览）
   2️⃣  UI 设计图片（静态视觉稿）
   3️⃣  两个都要
   ```

3. **HTML 原型模式**：

   生成独立的 HTML 文件，包含内联 CSS 和 JS，无需任何依赖即可打开：

   **技术选型：**
   - 使用 Tailwind CSS（CDN）快速布局
   - 使用原生 JS 实现交互（Tab 切换、弹窗、表单验证等）
   - 多页面用单文件 + hash 路由模拟

   **页面结构规范：**
   ```html
   <!DOCTYPE html>
   <html lang="zh-CN">
   <head>
     <meta charset="UTF-8">
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
     <title>[页面标题] - 原型</title>
     <script src="https://cdn.tailwindcss.com"></script>
     <!-- 内联自定义样式 -->
   </head>
   <body>
     <!-- 页面内容 -->
     <!-- 内联交互脚本 -->
   </body>
   </html>
   ```

   **必须包含的细节：**
   - 真实的占位数据（不要用 lorem ipsum，用中文模拟数据）
   - 所有按钮和链接要有 hover 效果
   - 表单要有基本的输入验证反馈
   - 列表/表格至少有 5 条模拟数据
   - Loading 状态、Empty 状态的展示
   - 响应式布局（如果用户要求）

4. **UI 图片模式**：

   使用图片生成工具，描述清楚：
   - 具体的页面布局和组件
   - 配色风格
   - 文字内容
   - 生成为 PNG 格式

5. 生成后提示用户：
   ```
   原型已生成！

   📁 文件：prototype/[name].html
   🌐 预览：在浏览器中打开即可查看

   需要调整什么？
   1️⃣  修改布局
   2️⃣  修改配色
   3️⃣  添加新页面
   4️⃣  添加更多交互
   ```

## 常用页面模板

快速生成时可参考以下模板方向：

**后台管理类：**
- 侧边栏导航 + 顶部栏 + 内容区
- Dashboard（数据卡片 + 图表 + 表格）
- CRUD 列表（搜索 + 表格 + 分页 + 操作按钮）
- 表单页（分步表单 / 单页表单）
- 详情页（基本信息 + 关联信息 + 操作日志）

**C 端/移动端类：**
- 登录/注册（账号密码 / 手机验证码 / 第三方登录）
- 首页/Feed 流
- 商品列表 / 搜索结果
- 详情页（图文 + 操作栏）
- 个人中心 / 设置页

**通用组件：**
- Modal 弹窗
- Toast 提示
- Drawer 抽屉
- Tab 切换
- 下拉菜单

## 注意事项

- HTML 原型必须是单文件，不依赖本地资源，打开即用
- 模拟数据要贴近真实业务场景，不要用 "测试1、测试2"
- 颜色方案默认使用中性色调，除非用户指定品牌色
- 移动端原型设置 max-width: 375px 并居中显示
- 如果用户后续要把原型转成真实代码，保持 HTML 结构语义化，便于迁移
- 每个页面的交互状态都要能演示（空状态、加载中、成功、错误）
