# refactor-helper

安全地重构代码，自动分析影响范围，保持功能不变。

## 输入

用户描述重构意图，例如：
- "把这个函数提取出来"
- "这个文件太大了，帮我拆分"
- "把所有 fetch 调用换成 axios"

## 输出

重构后的代码，附带变更说明和影响范围分析。

## 执行步骤

1. 理解重构目标，确认具体操作类型：
   - **提取**：提取函数 / 提取组件 / 提取常量 / 提取类型
   - **拆分**：拆分文件 / 拆分模块 / 拆分大函数
   - **重命名**：变量 / 函数 / 文件 / 类
   - **消除重复**：找出重复代码，抽象为通用函数
   - **迁移**：API 替换 / 库迁移 / 模式迁移
2. 分析影响范围：
   - 搜索所有引用该代码的位置
   - 列出需要同步修改的文件
   - 检查是否有测试需要更新
3. 展示重构计划：
   ```
   ## 重构计划

   **操作：** 将 UserService 拆分为 AuthService 和 ProfileService
   
   **影响范围：**
   - 需要修改 5 个文件
   - 需要更新 2 个测试文件
   - 需要更新 1 个导入文件

   **步骤：**
   1. 创建 AuthService，迁移认证相关方法
   2. 创建 ProfileService，迁移用户资料相关方法
   3. 更新所有导入引用
   4. 更新测试文件

   确认执行？
   ```
4. 用户确认后逐步执行，每步完成后展示 diff
5. 执行完成后运行测试确认没有破坏

## 注意事项

- 重构必须保持行为不变，不要在重构时加新功能或修 bug
- 大范围重构要分步进行，每步可独立验证
- 重构前确保有测试覆盖，如果没有，先建议补测试再重构
- 对于 TypeScript，利用类型系统确保重构后的类型安全
- 对于 Python，注意动态特性（getattr 等）可能导致静态分析遗漏引用
