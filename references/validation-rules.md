# 配置验证机制

## JSON Schema 验证

系统使用严格的 JSON Schema 验证技能配置，确保数据格式正确：

**必填字段验证：**
- `name`: 技能名称（3-50字符，小写字母数字连字符）
- `description`: 触发描述（50-300字符）

**可选字段验证：**
- `summary`: 技能简介（10-200字符）
- `dependencies`: 依赖库列表（格式验证）
- `steps`: 执行步骤（1-10个步骤）
- `notes`: 注意事项（最多10条）

**格式规范：**
```
✅ 正确格式：
- name: "pdf-creator"
- description: "当用户想要创建PDF文档时使用此技能..."
- dependencies: ["python-docx", "reportlab"]

❌ 错误格式：
- name: "PDF Creator"  # 包含大写字母和空格
- description: "创建PDF"  # 太短
- dependencies: ["python-docx", ""]  # 空字符串
```

## 自定义验证器

**描述质量验证：**
- 检查是否包含关键词：触发、关键词、场景
- 建议添加英文关键词提高触发准确性
- 推荐添加具体使用场景示例

**名称可用性验证：**
- 检查名称冲突（避免重复）
- 验证命名规范和可读性
- 建议更具描述性的名称

**依赖兼容性验证：**
- 检查已知依赖的版本兼容性问题
- 警告依赖过多的情况
- 提供兼容性建议

**步骤逻辑验证：**
- 验证步骤的完整性和逻辑性
- 检查是否包含明确的动作词汇
- 确保输入输出步骤的合理性

**内容一致性验证：**
- 检查描述和摘要的关键词一致性
- 验证输入输出的逻辑匹配
- 确保各部分内容的协调性

## 文件结构验证

**必需文件检查：**
- `SKILL.md`: 核心技能描述文件
- `README.md`: 使用说明文档

**SKILL.md 格式验证：**
- YAML frontmatter 完整性
- 必需章节存在性（输入、输出、执行步骤）
- Markdown 格式正确性

**目录结构一致性：**
- 检查声明的目录是否实际存在
- 验证 scripts/ 目录的 Python 包结构
- 确保文件引用的有效性

## 验证报告

验证完成后生成详细报告：

```
# my-skill 验证报告 - ✅ 通过

## 💡 建议 (可选改进)
1. 建议在描述中添加英文关键词，提高触发准确性
2. 建议至少包含2个步骤，描述完整的执行流程
3. 建议在步骤中使用明确的动作词汇

## ✅ 验证通过
技能配置符合所有规范，可以正常使用。
```

## 实时验证

**创建时验证：**
- 每完成一个步骤后进行实时验证
- 发现问题立即提示修复建议
- 阻止无效配置进入下一步

**编辑时验证：**
- 修改技能时实时验证变更
- 提供变更影响分析
- 确保修改不会破坏现有功能

**发布前验证：**
- 最终生成前的完整验证
- 生成详细的验证报告
- 确保技能包质量达标

## 验证级别

**错误级别（必须修复）：**
- Schema 验证失败
- 必需文件缺失
- 格式错误

**警告级别（建议修复）：**
- 描述质量问题
- 依赖兼容性风险
- 逻辑不一致

**建议级别（可选改进）：**
- 内容优化建议
- 最佳实践推荐
- 性能优化提示
